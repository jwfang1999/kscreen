K-Screen 1.0 Administrative Manual

The following is valid for K-Screen 1.0 beta

All tasks detailed in the manual assume you are log into K-Screen using an administrative account. See K-Screen 1.0 User/Client menu for non-administrative instructions.

Uploading Molecules

Refer to the kscreen_sdf2sql utility's README located at /kscreen /protected/ kscreen_bin/README on the server. The utility must be run locally due to the large size of typical SDF files.

Creating & Modifying Users

To create, go to Admin->Users->Create from the menu bar. Enter required information. To modify an existing user account, go to Admin->Users->View All. In the right-most column select the icon for what you want to do with the user (from left to right: View, Update, Delete).

Creating & Modifying Methods

Instructions are identical to Creating & Modifying Users except substitute 'Methods' for 'Users.'

Creating & Modifying Targets

Instructions are identical to Creating & Modifying Users except substitute 'Targets' for 'Users.'

Uploading or Updating a Primary Screen

To create a Primary Screen, must go to Admin->Primary Screens->Create. On this page you must provide three configuration and data files. Each of these files must obey a very strict format.

Configuration File

The configuration file is a grid-shaped CSV file designating the chemical compound in each well where each row of the CSV file is in the form of:

Plate Number, Plate Row (Single Character), Compound in Column 1, Compound in Column 2 ... Compound in Column 24

If a well has no compound, put 'E.' Positive and negative control wells should be marked 'P' and 'N' respectively. If no positive or negative wells are present, then you will not be able to use normalizations relying on positve and negative control.

Example: File:Sample Configuration.csv

Data File

The data file is similarly formatted. It is a CSV file designating the read value in each well where each row of the CSV file is in the form of:

Plate Number, Plate Row (Single Character), Read Value in Column 1,Read Value in Column 2 ... Read Value in Column 24

If a well has an invalid read-out put 'F.'

Example: File:Sample Data.csv

Permission File

The last file is the permission file. The permission file is a CSV file where each row assigns the a user plate level permissions for the primary screen. An empty default one

can be downloaded from the 'Create a Primary Screen' page. Te form of each row looks like:

User@Email, Condition 1, Condition 2 … Condition N

Where each Condition is one of the following:

  1. Plate Start–Plate End - User is allowed to see plates between Plate State and Plate End inclusive
  2. Plate Start- - User is allowed to see plates after Plate State including Plate Start
  3. -Plate End - User is allowed to see plates before Plate End including Plate End
  4. Plate Number - User is allowed to see Plate Number

Multiple conditions can be used when separated by commas. Be careful not to accidentally allow plates with conditions (2) and (3).

Example: File:Sample Permissions.csv

How the Permissions System Works

Permissions are assigned on a Primary Screen Daughter Plate level. These permissions determine if a user can view molecules on the Daughter Plate for both Primary Screens and Secondary Screens. This means that if a user does not have access to the plate with a molecule, then the user may not view the secondary screen associated with the molecule.

The system is somewhat unwieldy right now but we are looking into replacing it.

Primary Screen Submission

After creating these files (automated or manually), fill out the rest of the form and click submit. This launches a server side application which will process all your data. Give it at least 5 minutes and around 15 minutes for exceptionally large datasets (300 * 384 wells).

Go to Admin->Primary Screen->View All. This page will show the status of each submitted Primary Screen. Screens are color coded. Red means that the screen is being processed or has failed. Yellow means that the screen has not yet been published or is not visible. Green means that the screen has been published.

Primary Screen Presentation Configuration

Go to Admin->Primary Screen->View All. On the right-most column of the row corresponding to the Primary Screen you want to modify, select 'Presentation.' Here you can look at the various screen and plate level statistics for your screen. Make sure everything was processed correctly. If not, check that your configuration and data files conform to those specified in (V).

There are two sets of options to choose when configuring the Presentation of a Primary Screen. The first is visibility modifier. Primary Screens can have one of three settings:

  • Show -Primary Screen can be accessed by Clients from Screens menu.
  • Hide -Primary Screen cannot be accessed by Clients from Screens menu.
  • Secondary Screen Only - Similar to hide but allows Clients to view associated Secondary Screen data

[Place holder for Normalizations]

Click 'Submit.' K-Screen will now attempt to load the whole screen into the database. It may take a few minutes (less than 3) to complete. It will also report if there were any problems during the upload.

Updating a Primary Screen

Updating a Primary Screen is nearly identical to uploading a Primary Screen (V). The only difference is that it is accessed through Admin->Primary Screen->View All. On the right-most column of the row corresponding to the Primary Screen you want to modify, select 'Data.'

Uploading or Updating a Secondary Screen (Dose Response Curves)

The procedure to upload a Secondary Screen is much like that of uploading a Primary Screen (V). The differences are:

Select a Primary Screen instead of upload a Permissions File. Permissions are propagated for each curve depending on if the compound is visible to the user from the Primary Screen. The format for the Configuration File is changed (changes in bold) to the following:

Plate Number, Plate Row (Single Character), Compound in Column 1, Compound in Column 2 ... Compound in Column 24, Concentration in Column1, Concentration in Column 2 ... Concentration in Column 24

Example: File:Sample SScreen Configuration.csv

The Data File format remains unchanged. Fill out the rest of the form and click submit. Unlike Primary Screens, you can check progress % in the right-most column by going to Admin->Secondary Screen->View All.

Secondary Screen Presentation Configuration

Go to Admin->Secondary Screen->View All. On the right-most column of the row corresponding to the Secondary Screen you want to modify, select 'Presentation.' This should take you to the Curve Editor.

  1. On the left-side from top to bottom:
    1. Modifiers 'hide' and 'show' for toggling whether clients can access the screen.
    2. Update button for submitting the screen-level settings.
    3. Curve selection based on Molecule Name. As well as a checkbox left of each name for determining visiblity of the curve. Clicking on molecule name loads specific dose response curve data into other parts of the page.
  2. On the center-side from top to bottom:
    1. Dose response curve plot.
    2. Curve classification.
  3. On the right-side from top to bottom:
    1. Function Selection. Currently supported are Sigmoidal and Brain-Coussens curve models and a 'No Fit' class. Changing this drop-down menu will load the parameters for the specific model in the area below this drop-down menu.
    2. Function Parameters. This area displays function parameters based on the Function Selection drop-down menu. These can be modified and a new line can be generated by clicking the 'Custom Line' button.
    3. Dose Response Curve Points. This is a list of points. You can enable and disable points by clearing the checkbox next to each point. Not only does this mark point visually, it can be used to generate new best-fit lines by clicking the 'Best Fit' button. Custom Line, Best Fit, and Save buttons.
    4. Custom line and Best Fit buttons generate new lines described in (1.2) and (1.3) above.

The save button saves any changes made to the curve. You must click 'SAVE' to save your changes before switching to another curve.

When you are done updating all the curves, click 'Update.'

After processing is complete, if you return to the ‘View All’ page, you can now download a report of all curves and values.

Updating a Secondary Screen

Updating a Secondary Screen is nearly identical to uploading a Secondary Screen. The only difference is that it is accessed through Admin->Secondary Screen->View All. On the right-most column of the row corresponding to the Secondary Screen you want to modify, select 'Data.'

Reports

You may download a HTML document and curve images in a thumbnail to view a summary of all dose response curves by selecting 'reports' in the associated Secondary Screen row under Admin->Secondary Screen->View All.

Managing Attachments

On both Primary and Secondary ‘View All’ pages, there is an ‘Attachments(?)’ where ‘?’ is a number in the right most column. Selecting this will take you to an attachments menu where you can upload files that are accessible by those who have access to the particular screen.
