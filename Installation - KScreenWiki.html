Pre-requisites

The following is valid for K-Screen 1.0 beta

HTTP Server Installation

K-Screen was implemented and tested on the very popular Apache HTTP server from http://httpd.apache.org/. Windows IIS can theoretically be used by K-Screen but it remains untested on IIS configuraitons. Certain Linux flavors will come with Apache HTTP server pre-installed. Check with you administrator or server vendor if you are uncertain.

Apache HTTP server can be installed several ways. For Linux, you may be able to install via a online package for your particular flavor such as 'yum,' 'apt-get,' or which ever your Linux distribution is tied to.

To install Apache HTTP manually or on Windows, first go to http://httpd.apache.org/download.cgi. Download the package for your system configuration and follow the instructions to install.

HTTP Server Configuration

Find the httpd.conf file. Many default configurations on Linux put it in '/etc/httpd/conf'. It should be located in 'D:\Program Files\Apache Software Foundation\Apache2.2\conf\httpd.conf' on Windows if you installed it in the default location.

Configure the server by changing the 'Timeout' line in httpd.conf to something closer to 600 seconds or so (10 minutes) so long running processes do not fail. (marked in bold) 

   #
   # Timeout: The number of seconds before receives and sends time out.
   #
   Timeout 1200

You must also find your web root (htdocs) Directory settings. It should look something like the following, change the AllowOverride setting to 'all' from 'none.' (marked in bold) 

   #
   # This should be changed to whatever you set DocumentRoot to.
   #
   <Directory "/var/www/html">
   
   #
   # Possible values for the Options directive are "None", "All",
   # or any combination of:
   #   Indexes Includes FollowSymLinks SymLinksifOwnerMatch ExecCGI MultiViews
   #
   # Note that "MultiViews" must be named *explicitly* --- "Options All"
   # doesn't give it to you.
   #
   # The Options directive is both complicated and important.  Please see
   # http://httpd.apache.org/docs/2.2/mod/core.html#options
   # for more information.
   #
       Options Indexes FollowSymLinks
   
   #
   # AllowOverride controls what directives may be placed in .htaccess files.
   # It can be "All", "None", or any combination of the keywords:
   #   Options FileInfo AuthConfig Limit
   #
       AllowOverride All
   
   #
   # Controls who can get stuff from this server.
   #
       Order allow,deny
       Allow from all
   
   </Directory>

Run 'httpd –k restart' or otherwise restart the server for changes to take effect.

For those using IIS, similarly increase timeout time and be ready to make the K-Screen configuration folder server accessible only. (see below)

Mysql Database

K-Screen is currently configured to use a MySQL server by default and its chemical importer is specifically tied to MySQL. This will be fixed in future versions to work better with existing infrastructures. Until then, either install using a package manager or download and install using instructions found at http://dev.mysql.com/downloads/ or search for a package if using Linux.

MySQL Configuration

The MySQL configuration file is located at '/etc/my.cnf' for Linux or 'D:\Program Files\MySQL\MySQL Server #.#\my.ini' for Windows.

K-Screen allows other experimental results, supporting documents, billing, and other associated data to be uploaded as attachments with a limit of 50MB per item by default. You may either lower this in the K-Screen configuration file (see 'Configuring K-Screen') or add/set max_allowed_packet = 50M in my.cnf and restart the server. 

   [mysqld]
   max_allowed_packet = 50M
   datadir=/var/lib/mysql
   socket=/var/lib/mysql/mysql.sock

Additionally you should also create a new user for K-Screen (hence forth to be referred to as 'USER') with create, insert, update, delete permissions over the not yet created K-Screen database (hence forth referred to as 'KSCREEN_DATABASE_NAME'). You can create the database as well but that is not strictly required. 

   CREATE USER 'USER'@'localhost' IDENTIFIED BY 'PASSWORD';
   GRANT ALL ON KSCREEN_DATABASE_NAME.* TO 'USER'@localhost';

Refer to http://dev.mysql.com/doc/refman/5.1/en/create-user.html and http://dev.mysql.com/doc/refman/5.1/en/grant.html for further information.

PHP 5

PHP 5 packages and installations can be installed from http://php.net/manual/en/install.php or search for a package if using Linux. Be sure you install PHP 5 because K-Screen is not compatible with older versions of PHP.

You need to install:

  • All PDO libraries (http://www.php.net/manual/en/pdo.installation.php)
  • MySQL libraries (http://us.php.net/manual/en/mysql.installation.php)
  • GD2 (http://us.php.net/manual/en/image.installation.php)
  • Make sure that PHP DOM/XML is enabled(enabled by default: http://www.php.net/manual/en/dom.installation.php)

Most of these come pre-installed on Linux with PHP. You hopefully do not need to recompile php or anything drastic. In the Windows installer, you can tick the boxes by these libraries. However, you must follow some additional instructions in the associated links.

PHP 5 Configuration

Find the 'Resource Limits' section and change the settings there to resemble:

   max_execution_time = 600
   max_input_time = 600
   memory_limit = 2048M

If you do not do this, K-Screen's larger searches and batch processes will time out or run out of memory.

Yii Framework 1.1

Install Yii Framework 1.1 from http://www.yiiframework.com/download/. It should be installed in or as the root directory of your Apache HTTP server. Verify that it is indeed working by following the instructions in the Yii manual before proceeding.

R and R Libraries

The R Statistical language (at least version 2.10) is required by the K-Screen data processing. You may get this from http://www.r-project.org/ or search for a package if using Linux. Add R to the path.

You must also install several libraries:

  • lattice (http://cran.r-project.org/web/packages/lattice/index.html)
  • MASS (http://cran.r-project.org/web/packages/MASS/index.html)
  • drc (http://cran.r-project.org/web/packages/drc/index.html)
  • prada (http://www.bioconductor.org/packages/release/bioc/html/prada.html)

You may use the install.packages() command in R or obtain these from the CRAN/Bioconductor links. install.packages is simpler and recommended.

Installing K-Screen

To install K-Screen, open the root directory and copy 'kscreen' folder in the installation archive into the Yii Framework root directory. Make sure that you have permissions set in such a way that the web server can read, write, and execute in this directory. Also make sure you can access the webpage at 'localhost/(Yii Root Directory)/kscreen' on your corresponding web address. It is very important to determine that the 'kscreen/protected' path is inaccessible so sensitive data cannot be accessed. If you have not yet enabled .htaccess files, do so now. IIS users should make this folder web inaccessible. This directory should automatically be forbidden if .htaccess overwrites were enabled when configuring httpd.

K-Screen will not function as of yet because the K-Screen database is not configured.

Creating the MySQL Database and load Chemicals

To initialize your MySQL server with the K-Screen default schema and your chemical library, refer to the SDF2SQL README documentation in the ‘kscreen/protected/kscreen_bin/sdf2sql/’ directory. It will contain the instructions on how to create and load a SDF chemical database into K-Screen as well as create the database schema automatically.

Creating the MySQL Database without loading Chemicals

To initialize your MySQL server with the K-Screen default schema, open a command window and navigate to the ‘kscreen/protected/kscreen_bin/sdf2sql/’ directory. You can create a mysql database manually by logging into mysql with create permissions and entering (replace USER and KSCREEN_DATABASE_NAME by appropriate names): 

   create database KSCREEN_DATABASE_NAME;

Then use the mysql batch import routine using: 

   mysql –u USER –p KSCREEN_DATABASE_NAME < kscreenCleanSchema.sql

After entering USER’s password, this should automatically import the K-Screen schema into MySQL. K-Screen will have limited functionality until you load a chemical library however.

Configuring K-Screen

Navigate to and open ‘kscreen/protected/config/main.php.’ Change the settings object on line 163 starting with "'db'=>array(" to reflect your login and database settings. 

   'db'=>array(
       'connectionString' => 'mysql:host=localhost;dbname=KSCREEN_DATABASE_NAME',
       'emulatePrepare' => true,
       'username' => 'USER',
       'password' => 'PASSWORD',
       'charset' => 'utf8',
   ),

You may also change the maximum attachment upload size (inMB) specified at "Yii::app()->params->largestBlob" on line 45. 

   //Upload Settings
   Yii::app()->params->largestBlob = 50; //In MB
   Yii::app()->params->tempDir	    = 'tmp/';

Image locations (generated by SDF2SQL) are on line 13, this location must be somewhere web accessible. 

   //Molecule Image Data
   Yii::app()->params->imgDir = 'kscreen/bitmaps'; //location of images for molecules

Plate dimensions and coordinate systems are on line 115. 

   //Plate Coordinate System
   Yii::app()->params->plateNumCols = 24;
   Yii::app()->params->plateRows    = array('A','B','C','D','E','F','G','H','I','J','K','L','M','N','O','P');

Default Plate Configuration File

Typically, many labs use a standard mapping of compounds and wells. This file is used when creating new primary screens. You should modify this file or you may remove or rename this file to disable this feature. This file is located at 'kscreen/protected/kscreen_assets/default_plate_config.csv.' A 384 well configuration file is there as an example.

Rename the Admin User Account

K-Screen starts with a default user account called ‘admin@admin.admin’ with the password ‘admin.’ Log in as ‘admin@admin’ and change the account information. Otherwise, it would be very simple to bypass the security for K-Screen. This can be done by logging into Kscreen under this administrative account, going to the 'Admin' menu, selecting 'Users' and then finally 'View All.' This takes you to the user management page. In the right most column, select the middle (pencil/edit) icon, to change the settings for the default user.

Click this icon to edit the Default User.

K-Screen should be functional at this point.

Modifying CSS and Graphics

Graphics are stored in kscreen/images while css style sheets are located at kscreen/css

Updating K-Screen Components

K-Screen uses several major third party components. Updating these is not recommended because of compatibility issues. It is recommended that you keep the default versions unless there is a good reason to update.

JpGraph

JpGraph is used to generate charts and interactive browsers. You can download newer versions from http://jpgraph.net/download/ and replace the contents of the ‘kscreen/protected/kscreen_lib/graph/jpgraph’ directory with the contents of the jpgraph download.

Checkmol/ Matchmol and mol2ps by Norbert Haider

Obtain these executables from http://merian.pch.univie.ac.at/~nhaider/cheminf/cmmm.html and http://merian.pch.univie.ac.at/~nhaider/cheminf/mol2ps.html. Note that checkmol and matchmol are the same executables but have different functions depending on their name when called. Copy these to the corresponding directories in ‘kscreen/protected/kscreen_bin.’

Jmol

Obtain Jmol from http://jmol.sourceforge.net/download/. Extract *.jar files into ‘kscreen/applets/jmol.’
