Browsers

K-Screen contains browsers to (hopefully) allow for simple intuitive browsing of the information it stores.

Molecular Library

The molecular library browser allows users to view all the chemicals in the K-Screen library.

Molecule browser page showing search results and filter options

We see a tabular view of the molecule library uploaded to K-Screen. From left to right, we have a molecule preview thumbnail with a link to the more detailed molecule view including the chemical in a Jmol applet, fields representing the associated values for each molecule including Pubchem IDs, plate coordinates, supplier information and date added information.

This browser page also comes with a keyword filter option through the header. You may type any part of one of the column values and the browser will automatically try and display columns that match these keywords. For example if the 'name', 'KUID' in our example, column contained scientific names, typing in carbon in the text box provided below the column name would filter the results to show only names of molecules containing carbon like carbonic anhydrase. These filters also support logical operations. For example typing in >50 in the ID text box, you will only get molecules with IDs higher than 50. For alphabetic fields, this is done by alphabetical sort so typing in >50 in 'Pubchem ID' will display molecules with Pubchem IDs of 6, 7, 8, 50, 51 ...

Primary Screen

The primary screen browser allows the user to view the screen, plate, and well level statistics for each primary screen enabled for that particular user.

Primary Screen browser lets you view all levels of details for a screen

The page will start blank except for the left most box of the top right which is called 'Name.' This box lists all Primary Screens the user can view. Selecting a Primary Screen will generates all plates that the user is able to view. You may highlight multiple plates by holding down ctrl or shift. All plates should be highlighted by default. Click 'Search' to generate the results when this is done.

The components of the browser view are as follows:

  1. In the top right is the histogram of values across all selected plates to give the user an idea of the distribution of well activity values.
  2. Below the 'Name' and 'Plate(s)' selectors is the method and a link to the list of attachments (if any) associated with the primary screen.
  3. Below the selection area and the histogram is the screen level plate view. This view uses a box plot to show the distribution of activity values over each well of a plate by dividing the activity ranges into quartiles. Clicking on any of the box plot will update the plate level view.
  4. Below the screen level plate view and to the right are the plate heat map and the well activity plot. This view defaults to the lowest numbered plate selected.
    1. The well heat map shows shows positive and negative controls, failed/empty wells, and the relative activity levels of each player with values closer to blue low and values closer to red high. Clicking on a well will update the well view.
    2. The well activity plot is a scatter plot over all well activity values. Positive and Negative controls are marked yellow and green respectively. The x-axis is numbered 1 to 384 (in our example) with 1 to 24 being the first row, 25-48 being the second and so forth. Clicking on a data point will update the well view.
  5. To the left of the plate level view, we have the well level view. This shows the well's molecule in a JMol applet along with activity level information from other Primary Screen the user has access to. You may also launch structure searches from this window.

Secondary Screen (Dose Response Curves)

This browser allows users to choose dose response curves for viewing. Curve data along with molecule data and automated curve classification are all viewable.

Secondary Screen browser allows you to view Dose Response Curve information

Like the Primary Screen Browser, users select from successive selection boxes to pick a curve to view. Users first select a Primary Screen, which then allows the user to pick from the Secondary Screens which are associated with the Primary Screen. Lastly, the molecule name can be chooses to view the curve associated with that molecule.

The browser will then display the curve, curve classification (below the curve) and the curve data (left of the curve). Users can use this view to launch structural searches for the molecule. Curve data is displayed in text boxes and may be copied into files.

Searches

Library Batch Search

Library batch search is used to query data using a long list.

A file of or a copy/paste of a comma separated are both allowed. If multiple lists in different categories or files are uploaded, it will append the results of the first to the next... etc.

Both Mother and Daughter Plate coordinates can be searched using the * wildcard which can stand for anything. For example, 1*2 would return molecules at plate coordinate 1A2, 1B2, 1C2, 1D2... etc.

Primary Screen Searches

The search pages accessed under Molecule Library->Search Screens all share a common interface.

This interface allows for up to 5 (depending on K-Screen configuration) Primary Screens to be specified for each search. A range of activities may also be specified. When specifying multiple screens and ranges, K-Screen will search in the intersection of ranges. Thus, you should logically think of it as 'Molecules in Screen 1 between X and Y and Molecules in Screen 2 Between A and B.'

Text Searches

Keyword

Text search is a relatively straight forward search.

Each category accepts a single or multiple keyword(s) and will attempt to match the keywords. In the case of multiple keywords in a single category, the search will use logical OR. For example searching for "Chem1, Chem2" under 'Name(s)' (KUID(s) in the example) would return with results of Chem1 and Chem2. However, when multiple categories are used such as Name(s): "Chem1, Chem2" and D. Plate(s) "1, 2, 3" it would return any combination of Chem 1, Chem2 and 1, 2, 3.

Thus possible results would be a subset of:

  • Chem 1, D. Plate 1
  • Chem 1, D. Plate 2
  • Chem 1, D. Plate 3
  • Chem 2, D. Plate 1
  • Chem 2, D. Plate 2
  • Chem 2, D. Plate 3
Batch

Batch functions exactly like the library search batch mode except that it factors in Primary Screen activity range bounding.

Functional Group

Functional group searches returns molecules which classified as belonging to certain functional group.

You may select several though selecting more families will incur larger time penalties.

Structure

Structure searches use structural similarity to filter the molecular library.

You may access this page through the menu or through the Primary Screen, Secondary Screen browsers, or the Molecule View page. If you accessed it directly then you will need to draw a molecule in the JME ditor. Currently we support exact, substructure, and tanimoto similarity searches.

Third Party Searches

Currently we support interfaces to

  1. Pubchem
  2. ChemSpider
  3. ZINC

chemical databases.

Each of these searches functions very much like the structural similarity search.
